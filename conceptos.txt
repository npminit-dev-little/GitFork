Repositorio (Repository):
Un repositorio es un espacio donde se almacenan los archivos de un proyecto, junto con su historial de cambios. 
Es como una carpeta que contiene todos los archivos relacionados con tu proyecto, y cada cambio que realizas se registra en el historial del repositorio.

Commit:
Un commit es un registro de un cambio realizado en el repositorio. Cada vez que realizas una modificación en los archivos de tu proyecto, puedes hacer un commit para guardar esos cambios en el repositorio. 
Los commits generalmente van acompañados de un mensaje que describe los cambios realizados.

Branch (Rama):
Una rama es una versión paralela del repositorio principal (generalmente llamado "master" o "main"). 
Puedes crear ramas para trabajar en nuevas características o solucionar problemas sin afectar la rama principal. 
Una vez que los cambios en una rama están listos, pueden fusionarse con la rama principal.

Pull Request (Solicitud de extracción):
Un Pull Request es una solicitud que un desarrollador hace al propietario del repositorio para fusionar los cambios realizados en una rama con la rama principal. 
Permite revisar los cambios antes de fusionarlos, discutirlos y realizar ajustes si es necesario.

Fork:
Un fork es una copia de un repositorio en el que cualquier usuario puede trabajar de forma independiente. 
Los forks son útiles para probar cambios sin afectar al repositorio original y también para contribuir a proyectos de código abierto mediante la creación de nuevas características o correcciones y luego enviar Pull Requests al repositorio original.

Merge (Fusión):
La fusión es el proceso de combinar cambios de una rama a otra, generalmente fusionando los cambios de una rama secundaria en la rama principal. 
Esto se hace típicamente a través de un Pull Request.

Clone (Clonar):
Clonar un repositorio significa crear una copia local del repositorio remoto en tu propia máquina. 
Esto te permite trabajar en el proyecto de forma local y realizar cambios que luego puedes sincronizar con el repositorio remoto.

Issues (Problemas):
Los Issues son un lugar donde se pueden discutir problemas, errores, ideas y mejoras relacionadas con un proyecto. 
Pueden ser utilizados para rastrear tareas, informar de errores y colaborar en la resolución de problemas.

Wiki:
La Wiki es una sección del repositorio donde se pueden crear y mantener documentos de ayuda, guías de usuario, documentación técnica y cualquier otro tipo de contenido informativo relacionado con el proyecto.

GitHub Actions:
GitHub Actions es una característica que te permite automatizar flujos de trabajo dentro de tu repositorio. 
Puedes configurar acciones para que se ejecuten automáticamente en respuesta a eventos específicos, como la creación de Pull Requests o el push de código.
